docker简介

核心概念

  • 镜像（Image）：只读模板，包含应用和运行环境。
  • 容器（Container）：镜像的运行实例。
  • 仓库（Registry）：存储镜像的地方（如 Docker Hub）。
  • Dockerfile：文本文件，定义如何自动化构建镜像
  • 构建工具（DockerCompose）：通过 YAML 文件定义和管理多容器应用。
  • 数据卷（Volume）持久化容器数据，绕过联合文件系统。

核心组件

  • Docker 守护进程（dockerd）
    • 角色：服务端，负责管理容器、镜像、网络等。
    • 通信方式：通过 /var/run/docker.sock 或 TCP 端口（如 2375）接收客户端指令。
  • Docker 客户端（docker CLI）
  • 角色：用户与 dockerd 交互的工具。
  • 示例：docker build -t my-app . # 构建镜像（客户端发送指令给 dockerd）
  • Containerd
    • 角色：剥离自 Docker 的底层容器运行时，管理容器生命周期。
    • 架构：Docker CLI → dockerd → containerd → runc
  • runc
    • 角色：遵循 OCI 标准，实际调用 Linux 内核创建容器。
    • 作用：执行 namespaces + cgroups 等底层操作。

配置文件

Docker 服务端配置

  • /etc/docker/daemon.json

  • 作用：主配置文件，控制 dockerd 守护进程的核心行为。关键配置项：

    {
    “registry-mirrors”: [“https://<镜像加速地址>”], // 国内镜像加速
    “data-root”: “/mnt/docker-data”, // 修改默认存储路径
    “log-driver”: “json-file”, // 日志配置
    “log-opts”: {“max-size”: “10m”, “max-file”: “3”},
    “default-ulimits”: {“nofile”: {“Hard”: 65536, “Soft”: 32000}},
    “live-restore”: true // 容器在 dockerd 重启时保持运行
    }

  • 生效方式：
    sudo systemctl restart docker

  • 注意：

    配置好镜像加速之后，douck pull 会从新的镜像地址去拉，docker search 还是从原来的镜像地址去检索。阿里的镜像加速地址要在阿里云上使用，开源。镜像加速地址。{
    “registry-mirrors”: [
    “https://docker.m.daocloud.io“,
    “https://docker.imgdb.de“,
    “https://docker-0.unsee.tech“,
    “https://docker.hlmirror.com“,
    “https://cjie.eu.org“
    ]
    }

  • /lib/systemd/system/docker.service

  • 作用：Systemd 单元文件，定义 dockerd 的启动参数和依赖。

  • 关键内容：
    [Unit]
    Description=Docker Application Container Engine
    Requires=docker.socket
    After=network-online.target containerd.service

    [Service]
    ExecStart=/usr/bin/dockerd -H fd:// –containerd=/run/containerd/containerd.sock
    Restart=always
    LimitNOFILE=infinity # 资源限制

    [Install]
    WantedBy=multi-user.target

  • 自定义覆盖（推荐）：
    sudo systemctl edit docker # 生成 /etc/systemd/system/docker.service.d/override.conf

  • /lib/systemd/system/docker.socket

  • 作用：管理 Docker 的套接字激活（通过 /var/run/docker.sock 通信）。

  • 关键内容：
    [Socket]
    ListenStream=/var/run/docker.sock
    SocketMode=0660

Docker 客户端配置

  • ~/.docker/config.json
  • 作用：存储用户认证信息（如私有仓库登录凭证）和 CLI 默认行为。
  • 示例：
    {
    “auths”: {
    “https://registry.example.com“: {
    “auth”: “base64(username:password)”
    }
    },
    “credsStore”: “pass” # 使用系统密码管理器存储凭据
    }
  • 环境变量
  • 常用变量：
    export DOCKER_HOST=unix:///var/run/docker.sock # 指定 Docker 守护进程地址
    export DOCKER_BUILDKIT=1 # 启用 BuildKit 构建引擎

容器/镜像构建配置

  • Dockerfile
  • 核心指令：
    FROM ubuntu:22.04
    RUN apt-get update && apt-get install -y curl
    COPY ./app /app
    EXPOSE 8080
    CMD [“/app/start.sh”]
  • dockerignore
    • 作用：排除构建时不需要的文件。
    • 示例：
      .git
      node_modules/
      *.log

多容器编排配置

  • docker-compose.yml

    • 示例：
      version: ‘3.8’
      services:
      web:
      build: .
      ports: [“8000:8080”]
      volumes: [“./code:/app”]
      redis:
      image: redis:alpine

  • docker-compose.override.yml

    • 作用：开发环境专用配置（如挂载调试工具）。

    • 示例：
      version: ‘3.8’
      services:
      web:
      environment:
      DEBUG: “true”
      volumes:

      “/dev/log:/dev/log” # 挂载系统日志

配置文件逻辑关系与优先级

  • 启动流程
    Systemd 加载 docker.socket → 激活 docker.service → 启动 dockerd。
    dockerd 读取 /etc/docker/daemon.json 应用全局配置。
    用户通过 CLI 或 Compose 操作时，参考 ~/.docker/config.json 和环境变量。
  • 配置优先级
    从高到低：
    命令行参数 > docker-compose.yml > daemon.json > docker.service 默认值
    例如：docker run –log-driver=syslog 会覆盖 daemon.json 中的日志配置。
  • 关键路径总结
    文件/目录 作用
    /etc/docker/daemon.json Docker 守护进程主配置
    /lib/systemd/system/docker.service Systemd 服务定义（启动参数）
    ~/.docker/config.json 客户端认证和默认行为
    Dockerfile 定义镜像构建步骤
    docker-compose.yml 多容器编排声明

数据管理

  • 数据卷（Volume）
    • 作用：持久化容器数据，绕过联合文件系统。
    • 操作：docker volume create my-vol # 创建卷
      docker run -v my-vol:/data # 挂载卷
  • 绑定挂载（Bind Mount）
  • 作用：将主机目录直接挂载到容器。
  • 示例：docker run -v /host/path:/container/path nginx

底层技术

  • 命名空间（Namespaces）
    • 作用：实现资源隔离（进程、网络、文件系统等）。
    • 示例：PID Namespace：容器内进程独立于主机进程树。
      Network Namespace：容器拥有独立 IP 和端口。
  • 控制组（cgroups）
  • 作用：限制容器资源使用（CPU、内存、磁盘 I/O）。
  • 示例：docker run –cpus=2 –memory=500m nginx # 限制 2 核 CPU 和 500MB 内存
  • 联合文件系统（UnionFS）
  • 作用：支持镜像分层存储，如 overlay2、aufs。
  • 原理：只读层（镜像） + 可写层（容器） = 最终文件系统视图。
