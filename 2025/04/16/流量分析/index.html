流量分析

发表于2025-04-16|更新于2025-04-22
|浏览量:

简介

  • 网卡：wireshark是对主机网卡上的数据流是进行抓取

  • 网卡模式

    • 混杂模式:不管目的是否是自己,都接收
    • 非混杂模式:默认情况下,主机的网卡处于此模式,不会接收目的非自己的数据

  • 两种过滤器

    • 捕获过滤器:在抓包之前先进行过滤(只抓某种类型的包或者不抓某些类型的包)
    • 显示过滤器:抓包前后抓包后都可以进行过滤,但是不会影响抓取的包(会抓取所有的包,只不过在查看的时候只显示某些包

  • 捕获过滤器

    • 语法

      • 类型:host net port
      • 方向:src dst
      • 协议:ether ip tcp udp http…
      • 逻辑运算符:&&与 ||或 !非

    • 举例

      • 抓取源IP为192.168.18.14并且目标端口为80的报文src host 192.168.18.14 && dst port 80
      • 抓取IP为192.168.18.14或者IP地址为192.168.18.1 host 192.168.18.14 || host 192.168.18.1
      • 不抓取广播包 ! broadcast
      • 抓取源IP为192.168.18.14或者源192.168.18.0/24,目的TCP端口号在200到1000之间,并且目的位于129.0.0.0/8 (src host 192.168.18.14 || src net 192.168.18.0/24)&&(dst port range 200-10000 && dst net 129.0.0.0/8)

层次分析

1. Frame（帧）

  • 所属层次：物理层/数据链路层（OSI L1/L2）。

  • 作用：表示捕获的原始数据帧，包含实际传输的二进制数据。

    关键字段

    • Frame Number：数据包在捕获文件中的序号。
    • Time：捕获时间戳（相对/绝对时间）。
    • Length：帧的总长度（字节）。
    • Protocols：帧内封装的协议类型（如 Ethernet、IP、TCP等）。

    示例

2. Ethernet（以太网头部）

  • 所属层次：数据链路层（OSI L2）。

  • 作用：负责本地网络（如 LAN）内的设备间数据传输。

    关键字段

    • Source MAC：发送方的物理地址（如 00:1a:2b:3c:4d:5e）。
    • Destination MAC：接收方的物理地址（如广播地址 ff:ff:ff:ff:ff:ff）。
    • Type：上层协议类型（如 0x0800 表示 IPv4，0x86DD 表示 IPv6）。

    示例

3. Internet Protocol（IP 头部）

  • 所属层次：网络层（OSI L3）。

  • 作用：实现跨网络的逻辑寻址和路由。

    关键字段

    • Source IP：发送方 IP 地址（如 192.168.1.1）。
    • Destination IP：接收方 IP 地址（如 10.205.125.1）。
    • Protocol：上层协议（如 6 表示 TCP，17 表示 UDP）。
    • TTL：生存时间（防止数据包无限循环）。

    示例

4. Transmission（传输层协议，如 TCP/UDP）

  • 所属层次：传输层（OSI L4）。

  • 作用：管理端到端的通信（可靠性、流量控制等）。

    常见协议及字段

    TCP（传输控制协议）

    • Source Port / Destination Port：应用程序端口（如 80 为 HTTP）。
    • Sequence/Acknowledgement Number：保证数据有序和可靠传输。
    • Flags：控制字段（如 SYN、ACK、FIN）。

    UDP（用户数据报协议）

    • 更简单，无连接，仅有端口号和校验和。

协议栈的完整关系（自上而下）：

  • Frame 是原始数据，Ethernet 解决“下一跳给谁”，IP 解决“最终发给谁”，TCP/UDP 解决“哪个应用处理”。
  • 在 Wireshark 中，可以通过展开每个层查看详细字段，或使用过滤表达式（如 tcp.port == 443）定位特定流量。

NTP协议报文

NTP（Network Time Protocol，网络时间协议）用于在计算机网络中同步各设备的系统时间，基于 UDP 123 端口 工作。主要功能

  • 提供高精度的时间同步（理论上可达毫秒级甚至更高）。

  • 采用分层（Stratum）架构，防止时间同步环路。

  • 支持多种时间源（如 GPS、原子钟、其他 NTP 服务器）。

NTP 支持多种工作模式，常见的有：

客户端-服务器模式（Client-Server）

  • 客户端（Mode=1）向服务器（Mode=2）请求时间同步。
  • 服务器返回时间戳（Origin/Receive/Transmit Timestamp）。

对等模式（Peer-to-Peer）

  • 两个 NTP 服务器互相校准时间（Mode=4）。

广播/多播模式（Broadcast/Multicast）

  • 服务器（Mode=3）周期性广播时间，客户端被动接收。

ARP协议报文

作用：通过 ​​IP地址​​ 查询对应的 ​​MAC地址​​，实现局域网（LAN）内设备通信。

核心特点：

  1. 工作层级：工作在 数据链路层（L2），但为 网络层（L3） 服务。
  2. 广播请求：ARP请求以广播形式发送（目标MAC = FF:FF:FF:FF:FF:FF）。
  3. 单播响应：只有目标主机回复ARP响应，携带自己的MAC地址。
  4. 缓存机制：设备会缓存ARP表（arp -a 可查看），减少重复查询。

典型场景：

  • 主机A（IP1）想和主机B（IP2）通信，但不知道B的MAC地址，于是发送ARP请求。
  • 主机B收到后，回复ARP响应，告知自己的MAC地址。

安全问题：

  • ARP欺骗（ARP Spoofing）：攻击者伪造ARP响应，劫持流量（需防范）。
